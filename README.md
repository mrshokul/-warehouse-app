# ฟ้าตระการมอลล์ — ระบบบริหารสินค้าในโกดัง (WMS)

ระบบติดตามสินค้าและความรับผิดชอบร่วมในโกดัง สร้างจากเอกสาร SRS — เน้น **ป้องกันทุจริต / รับรู้ด้วยกัน / ไม่โทษคนเดียว** ผ่านหลักการ "สองมือเสมอ"

เทคโนโลยี: Node.js + Express + SQLite (ไม่ต้องตั้งเซิร์ฟเวอร์ฐานข้อมูลแยก) · หน้าบ้าน HTML/CSS/JS ล้วน (ไม่มีขั้นตอน build)

---

## การติดตั้งและรัน

ต้องมี **Node.js เวอร์ชัน 18 ขึ้นไป** ติดตั้งในเครื่องก่อน

```bash
# 1) ติดตั้ง dependency
npm install

# 2) เตรียมฐานข้อมูลตั้งต้น (สร้างช่อง + นำเข้าสินค้า 12,166 รายการ + ผู้ใช้ตัวอย่าง)
npm run seed

# 3) เปิดเซิร์ฟเวอร์
npm start
```

แล้วเปิดเบราว์เซอร์ไปที่ **http://localhost:3000**

> ถ้าพอร์ต 3000 ชนกับโปรแกรมอื่น สั่ง `PORT=4000 npm start` แล้วเปิด http://localhost:4000

## บัญชีผู้ใช้

**บัญชีหลัก (คนแรกที่เข้าระบบได้):**

| ชื่อผู้ใช้ | รหัสผ่าน | บทบาท |
|---|---|---|
| `admin` | `admin045` | 👑 เทพเจ้าสูงสุด — ทุกอย่าง + จัดการผู้ใช้ |

**ผู้ใช้ตัวอย่างไว้ทดสอบ (รหัสผ่าน `1234`):**

| ชื่อผู้ใช้ | บทบาท | ทำอะไรได้ |
|---|---|---|
| `guardian` | ⚔️ เทพพิทักษ์ | ทำรายการ + อนุมัติ + ลบสินค้า |
| `angel1` | 🧚 นางฟ้า | สร้างสินค้า/รายการ + ยืนยันของคนอื่น |
| `angel2` | 🧚 นางฟ้า | เหมือน angel1 (ไว้ทดสอบกฎสองมือ) |
| `intern` | 🌱 มนุษย์ฝึกหัด | ดูอย่างเดียว |

> เข้าระบบครั้งแรกด้วย `admin` แล้วใช้เมนู "จัดการผู้ใช้" เพิ่มพนักงานจริง · **เปลี่ยนรหัสผ่าน admin และลบผู้ใช้ตัวอย่างก่อนใช้งานจริง**

### ลองกฎสองมือ
1. เข้าด้วย `angel1` → เมนู "สร้างรายการ" → นำเข้าสินค้าเข้าช่อง เช่น A01 → บันทึก (รออนุมัติ)
2. ออกจากระบบ เข้าด้วย `angel2` → เมนู "รออนุมัติ" → กดยืนยัน → ของจึงเข้าสต็อกจริง
3. ถ้าลองให้ `angel1` ยืนยันรายการที่ตัวเองสร้าง ระบบจะปฏิเสธ

## ฟีเจอร์ที่ทำแล้ว (กลุ่ม "ต้องมี")

- เข้าสู่ระบบ + สิทธิ์ตามบทบาท 4 ระดับ (บังคับในระดับเซิร์ฟเวอร์)
- ฐานข้อมูลสินค้า 12,166 รายการ (รหัส=บาร์โค้ด ห้ามซ้ำ; สินค้าใหม่ที่ไม่มีบาร์โค้ด พนักงานกำหนดรหัสเอง ระบบเช็คซ้ำให้)
- สร้างรายการ นำเข้า / เบิกออก / ย้ายช่อง / นับสต็อกตั้งต้น — ทุกรายการเข้าคิว "รออนุมัติ"
- **กฎสองมือ**: ผู้ยืนยันต้องไม่ใช่ผู้สร้าง (บังคับฝั่งเซิร์ฟเวอร์ เลี่ยงไม่ได้)
- หน้ารายการรออนุมัติ → ยืนยัน / ยกเลิก (ยกเลิกต้องระบุเหตุผล)
- แก้ไขรายการที่ยืนยันแล้ว → ต้องมีคนยืนยันการแก้ + เก็บประวัติ (ค่าเก่า→ค่าใหม่) + ปรับสต็อกตามส่วนต่าง
- ดูสต็อกปัจจุบัน (ของอยู่ช่องไหน เหลือเท่าไหร่ + ยอดรวมต่อสินค้า)
- รายงานย้อนหลัง (รายสินค้า / รายช่อง / และกรองผู้ทำได้)
- จัดการผู้ใช้ (เพิ่ม/ระงับ/กำหนดบทบาท) — เฉพาะเจ้าของร้าน

## ยังไม่ทำ (ต่อยอดได้)
- อัปโหลดรูปภาพสินค้า (ตอนนี้รองรับเก็บ URL รูปในฐานข้อมูลแล้ว เหลือทำหน้าอัปโหลด)
- รายงานรายคนแบบหน้าจอเฉพาะ (ข้อมูลมีครบใน API แล้ว)
- การจัดมาตรฐานหน่วยนับเชิงลึก (ตอน seed ทำเบื้องต้น: ตัดเว้นวรรค, แก้ เช็ท→เซ็ท, ตัดค่า `4.5` ออก 1 รายการ)
